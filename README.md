# Signage Ctrl — Digital Signage Platform

A simple digital signage web app to control multiple TVs from a phone or browser. Login with the same account on the TV and Controller, choose the TV PIN, upload images/videos, and the selected TV updates automatically.

---

## Features

- Sign up and login with Supabase Auth
- TV User mode for full-screen display
- Controller mode for phone/browser control
- Control up to 10 TVs using TV PINs
- Same account + same TV PIN syncs the Controller with the TV
- Different accounts are isolated and cannot control each other’s TVs
- Upload images/videos to Supabase Storage
- Store TV state in Supabase Database
- Mobile responsive controller UI
- TV switching from Controller using the TVs tab
- Full-screen media display on TV

---

TV Browser / Smart TV

↑

Controller Mode
↓ Select same TV PIN
Upload image/video
↓
Supabase Storage
↓

↓

## Important Sync Logic

The app syncs TVs using:

user_id + tv_id

That means:

Same account + TV1 PIN = controls that account’s TV1  
Different account + TV1 PIN = separate TV1 state

Example:

TV device:
Login: user1@gmail.com
Mode: TV User
PIN: 1111
Result: TV1 display waiting

Phone controller:
Login: user1@gmail.com
Mode: Controller
PIN: 1111
Result: Controls TV1

Different account example:

TV device:
Login: user1@gmail.com
PIN: 1111

Phone controller:
Login: user2@gmail.com
PIN: 1111

Result:
Does not control user1 TV1.
It creates/uses user2 TV1 separately.

## Supabase Setup

### 1. Create Supabase Project

Create a new Supabase project.

Copy these values:

Project URL  
Anon public key

Use them in:

const SUPABASE_URL = 'YOUR_SUPABASE_PROJECT_URL';
const SUPABASE_KEY = 'YOUR_SUPABASE_ANON_PUBLIC_KEY';

Do not use the service role key in frontend code.

---

## 2. Enable Email Auth

In Supabase:

Authentication → Providers → Email → Enable

For testing, you can turn off email confirmation:

Authentication → Providers → Email → Confirm email OFF

---

## 3. Create Storage Bucket

In Supabase:

Storage → New bucket

Bucket name:

signage-media

Set bucket as public:

Public bucket = ON

---

## 4. Create tv_states Table

Run this in Supabase SQL Editor:

create table if not exists tv_states (
  id bigint generated by default as identity primary key,
  user_id uuid not null,
  tv_id text not null,
  layout_id text,
  cells jsonb default '[]'::jsonb,
  updated_at bigint
);

create unique index if not exists tv_states_user_tv_unique
on tv_states(user_id, tv_id);

If you already had old shared data, clear it:

delete from tv_states;

---

## 5. Enable RLS for tv_states

Run this in Supabase SQL Editor:

alter table tv_states enable row level security;

create policy "Users can read own tv states"
on tv_states
for select
to authenticated
using (auth.uid() = user_id);

create policy "Users can insert own tv states"
on tv_states
for insert
to authenticated
with check (auth.uid() = user_id);

create policy "Users can update own tv states"
on tv_states
for update
to authenticated
using (auth.uid() = user_id)
with check (auth.uid() = user_id);

---

## 6. Storage Policies

Run this in Supabase SQL Editor:

create policy "Allow authenticated uploads"
on storage.objects
for insert
to authenticated
with check (bucket_id = 'signage-media');

create policy "Allow authenticated updates"
on storage.objects
for update
to authenticated
using (bucket_id = 'signage-media')
with check (bucket_id = 'signage-media');

create policy "Allow public reads"
on storage.objects
for select
to public
using (bucket_id = 'signage-media');

If a policy already exists, Supabase may show an error. That is okay if the policy is already created.

## Common Issues

### Upload failed

Check:

1. Bucket name is signage-media
2. Bucket is public
3. Storage policies exist
4. User is logged in
5. File size is not too large

### Invalid API key

Check:

SUPABASE_URL and SUPABASE_KEY must belong to the same Supabase project.  
Use anon public key only.

### Email not confirmed

Either confirm email from inbox or turn off confirmation for testing:

Authentication → Providers → Email → Confirm email OFF

### Different accounts controlling same TV

This should not happen if user_id + tv_id logic is active.

Confirm tv_states has:

user_id  
tv_id

and unique index:

create unique index if not exists tv_states_user_tv_unique
on tv_states(user_id, tv_id);

### Vercel building old commit

Check latest commit:

git log --oneline -3

Trigger fresh deploy:

git commit --allow-empty -m "trigger vercel deploy"
git push

Then check Vercel build log commit hash.

- Do not commit Supabase service role key.
- Frontend can use Supabase anon public key.
- For production, keep RLS policies enabled.
- Keep uploads restricted to authenticated users.
